# نظام إدارة المنتجات المباشر 📦

تطبيق ويب بسيط وفعال لإدارة المنتجات والمخزون مع التحديث المباشر، يشبه Google Sheets ولكن مخصص لإدارة المنتجات.

## ✨ المميزات

### 🔥 التحديث المباشر (Real-time)
- مشاهدة التغييرات فورياً بدون إعادة تحميل الصفحة
- تزامن البيانات بين جميع المستخدمين والأدمن

### 👥 واجهتان منفصلتان
1. **واجهة المستخدمين**: لإدخال البيانات بطريقة سهلة ومرتبة
2. **لوحة تحكم الأدمن**: لمراقبة وإدارة جميع البيانات

### 📊 إدارة البيانات
- **اسم المنتج**: وصف واضح للمنتج
- **كود المنتج**: معرف فريد لكل منتج
- **كود الفرع**: تحديد الفرع المسؤول
- **مقياس الاحتياج**:
  - مطلوب جداً (عاجل)
  - مطلوب (عادي)
  - مطلوب بتاريخ لا يقل عن 3 أشهر
  - جديد

### 🎨 تصميم متجاوب وجميل
- تصميم عصري وألوان مريحة للعين
- متوافق مع جميع الأجهزة (كمبيوتر، تابلت، موبايل)
- رموز واضحة وسهلة الفهم

### 🔧 أدوات الأدمن المتقدمة
- إحصائيات مباشرة لكل مستوى احتياج
- فلترة وبحث متقدم
- ترتيب البيانات حسب أي عمود
- تصدير البيانات إلى ملف CSV
- حذف المنتجات أو جميع البيانات

## 🚀 طريقة الإعداد

### 1. إعداد Firebase
1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. أنشئ مشروع جديد
3. فعّل "Realtime Database"
4. اختر "Start in test mode" للقواعد
5. انسخ إعدادات المشروع

### 2. تحديث إعدادات Firebase
افتح ملف `firebase-config.js` وحدث القيم التالية:

```javascript
const firebaseConfig = {
    apiKey: "your-api-key-here",
    authDomain: "your-project.firebaseapp.com",
    databaseURL: "https://your-project-default-rtdb.firebaseio.com/",
    projectId: "your-project-id",
    storageBucket: "your-project.appspot.com",
    messagingSenderId: "123456789",
    appId: "your-app-id"
};
```

### 3. تشغيل التطبيق
```bash
# إذا كان لديك Python
python -m http.server 8000

# أو إذا كان لديك Node.js
npx serve .

# أو إذا كان لديك Live Server في VS Code
# اضغط بالزر الأيمن على index.html واختر "Open with Live Server"
```

### 4. الوصول للتطبيق
- **الصفحة الرئيسية**: `http://localhost:8000`
- **لوحة تحكم الأدمن**: `http://localhost:8000/admin`

## 🎯 كيفية الاستخدام

### للمستخدمين:
1. افتح الصفحة الرئيسية
2. املأ بيانات المنتج في النموذج
3. اختر مستوى الاحتياج من القائمة المنسدلة
4. اضغط "حفظ المنتج"
5. ستظهر البيانات فوراً في الجدول أسفل النموذج

### للأدمن:
1. افتح لوحة تحكم الأدمن
2. شاهد الإحصائيات المباشرة في الأعلى
3. استخدم المرشحات للبحث والتصفية
4. رتب البيانات بالضغط على عناوين الأعمدة
5. صدّر البيانات أو احذف منتجات معينة

# نظام دخول المدراء والمستخدمين

## نظرة عامة
نظام دخول متكامل يدعم إدارة المدراء والمستخدمين مع إعدادات أمان متقدمة وإدارة الجلسات.

## المتطلبات
- متصفح ويب حديث
- ملف `.env` مُعد بشكل صحيح

## إعداد متغيرات البيئة

### 1. إنشاء ملف .env
انسخ الملف `.env.example` إلى `.env` وقم بتعديل القيم حسب احتياجاتك:

```bash
cp .env.example .env
```

### 2. متغيرات بيانات المدراء
```bash
# Admin Credentials
ADMIN1_USERNAME=admin1
ADMIN1_PASSWORD=admin123
ADMIN1_ROLE=super_admin
ADMIN1_NAME=مدير النظام الرئيسي

ADMIN2_USERNAME=admin2
ADMIN2_PASSWORD=admin456
ADMIN2_ROLE=admin
ADMIN2_NAME=مدير فرعي

ADMIN3_USERNAME=admin3
ADMIN3_PASSWORD=admin789
ADMIN3_ROLE=moderator
ADMIN3_NAME=مشرف
```

### 3. إعدادات الأمان
```bash
# Security Settings
MAX_LOGIN_ATTEMPTS=3           # عدد محاولات الدخول المسموحة
LOCKOUT_TIME_MINUTES=15        # مدة القفل بالدقائق
SESSION_TIMEOUT_MINUTES=60     # مدة انتهاء الجلسة بالدقائق
```

### 4. إعدادات التطبيق
```bash
# App Settings
MAX_USERS=40                   # العدد الأقصى للمستخدمين
APP_ENV=development            # بيئة التطبيق
```

## ميزات الأمان المُطبقة

### 1. حماية من محاولات الدخول المتكررة
- قفل الحساب بعد عدد محدد من المحاولات الفاشلة
- مدة قفل قابلة للتخصيص
- عرض الوقت المتبقي للقفل

### 2. إدارة الجلسات
- انتهاء تلقائي للجلسة بعد فترة عدم النشاط
- إعادة تعيين وقت الجلسة عند النشاط
- تنبيه قبل انتهاء الجلسة

### 3. فصل البيانات الحساسة
- جميع كلمات المرور في ملف `.env`
- عدم تضمين البيانات الحساسة في الكود المصدري
- حماية ملف `.env` من التتبع في Git

## الاستخدام

### دخول المستخدمين
- أسماء المستخدمين: `user01` إلى `user40`
- كلمات المرور: `pass1` إلى `pass40`

### دخول المدراء
- يتم تحديد بيانات المدراء من ملف `.env`
- أدوار مختلفة: `super_admin`, `admin`, `moderator`

## الأمان في الإنتاج

### 1. تشفير كلمات المرور
```javascript
// استخدم bcrypt أو مكتبة تشفير مناسبة
const bcrypt = require('bcrypt');
const hashedPassword = await bcrypt.hash(password, 10);
```

### 2. HTTPS
- استخدم دائماً HTTPS في الإنتاج
- تأكد من صحة شهادات SSL

### 3. CSP (Content Security Policy)
```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'">
```

### 4. متغيرات البيئة الآمنة
- لا تضع البيانات الحساسة في الكود
- استخدم خدمات إدارة الأسرار للإنتاج
- قم بتدوير كلمات المرور دورياً

## استكشاف الأخطاء

### المشاكل الشائعة
1. **لا يمكن تحميل الإعدادات**: تأكد من وجود ملف `.env`
2. **فشل تسجيل الدخول**: تحقق من صحة بيانات `.env`
3. **انتهاء الجلسة المبكر**: اضبط `SESSION_TIMEOUT_MINUTES`

### السجلات
- جميع محاولات الدخول يتم تسجيلها
- النشاطات محفوظة في `localStorage`
- راجع وحدة تحكم المتصفح للأخطاء

## المساهمة
1. تأكد من عدم رفع ملف `.env`
2. استخدم `.env.example` للتوضيح
3. اتبع معايير الكود المُعرفة
